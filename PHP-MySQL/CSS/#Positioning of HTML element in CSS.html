POSITIONING OF HTML ELEMENT

Why positioning is important?

Because....
  • By default, HTML element appears on the web page in the same sequence as they are in the HTML-code.
  • So that, HTML is not capable to set position of HTML-element with precision.

Position-Property

The position property specifies the types of POSITIONING.

It has 4-types.

  1. Position-Static; //By-Default//
  2. Position-fixed;
  3. Position-relative;
  4. Position-absolute;

POSITIONING

  • Elements are then positioned using top,bottom,left and right property.
  • #Those properties will not work, unless the position-property is set first(value should not be static).
  • They also work differently depending on the position value.

1.Static

  • HTML elements are positioned static by-default.
  • So that, top,bottom,left and right property will not work here.
  • Element will appear, according to the normal flow.

2.Fixed

  • Element with position-fixed always stays in the same place, even if the page is scrolled.
  • top,bottom,left and right property will work.
  • It can be set as....position: fixed;

3.relative

  • An element with position: relative; is positioned relative to it's normal position.
  • Top,right,bottom,left will work.

4.Absolute

  • A positioned element is one whose position is anything except static.
  • An Absolute-positional element is positional, relative to the nearest positional-Ancestor.
  • If an Absolute-positional element has no positioned-Ancestors, it uses the document body, and moves along with the page scrolling.

# Z-Index

  • When element are positioned, they can overlaps other elements.
  • The Z-index property specifies the stack order of an element.
  • An element With greater stack order is always in front of an element with a lower stack order.
  • If two positioned element overlaps without a Z-index specified, the element positioned last in the HTML code, will be shown on Top.
  • By-Default the value of Z-index is zero.
